\## Project Overview

This project demonstrates REST API automation testing using Postman for the Swagger PetStore application.  

It covers end-to-end API validation with positive, negative, and regression scenarios.



\## Application Under Test

Swagger PetStore (Public REST API)



\## Modules Covered

\- Pet Module

\- Store Module

\- User Module



\## Tools \& Technologies

\- Postman

\- REST API

\- JavaScript (Postman Test Scripts)

\- Swagger / OpenAPI

\- Newman (CLI)



\## Test Coverage

\- CRUD operations (POST, GET, PUT, DELETE)

\- Positive and Negative test cases

\- Regression API testing

\- Status code validation

\- Response time validation

\- Header and response body validation

\- Error handling (404, invalid inputs, incorrect payloads)



\## How to Run Tests

1\. Import the Postman collection and environment

2\. Select the environment

3\. Execute tests using Postman Runner or Newman CLI



\## Test Execution Report

A sample Newman HTML execution report is available under the `reports` folder.



\## Key Highlights

\- Automated 150+ REST API test cases

\- Reduced manual API testing effort by ~70%

\- Implemented reusable collection-level scripts

\- Used environment variables for dynamic data handling
